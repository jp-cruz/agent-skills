CONFIGURATION FLOW
==================

.env.example
    ↓ (copy & customize)
.env
    ↓ (read by docker-compose)
docker-compose.yml
    ↓ (substitutes variables)
Dockerfile
    ↓ (builds)
thoth:latest (docker image)
    ↓ (docker-compose up)
thoth-app (running container)


ENVIRONMENT VARIABLES SUPPORTED
================================

THOTH_PORT              Default: 8080
THOTH_DATA_DIR          Default: ./thoth-data
THOTH_WORKSPACE_DIR     Default: ./thoth-workspace
OLLAMA_BASE_URL         Default: http://host.docker.internal:11434
RESTART_POLICY          Default: unless-stopped


CROSS-PLATFORM SUPPORT
======================

✅ macOS (Intel & Apple Silicon)
   └─ Uses: host.docker.internal for Ollama access
   └─ Setup: ./setup.sh

✅ Windows (WSL 2 Backend)
   └─ Uses: host.docker.internal for Ollama access
   └─ Setup: setup.bat (from PowerShell or Git Bash)

✅ Linux (host or remote Ollama)
   └─ Uses: Local IP or remote host IP for Ollama
   └─ Setup: ./setup.sh (detects automatically)

QUICK START
===========

macOS/Linux:
  $ ./setup.sh              # Validates prerequisites, creates .env
  $ vim .env                # Customize if needed
  $ docker-compose up -d    # Start
  $ open http://localhost:8080

Windows (PowerShell):
  > setup.bat               # Validates prerequisites, creates .env
  > notepad .env            # Customize if needed
  > docker-compose up -d    # Start
  > start http://localhost:8080


COMMON COMMANDS
===============

docker-compose up -d           Start in background
docker-compose down            Stop and remove container
docker-compose logs -f         View live logs
docker-compose ps              Check status
docker-compose exec thoth bash Open shell inside container
docker-compose up --build      Rebuild and start
