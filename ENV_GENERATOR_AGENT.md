# Automated .env Generator — Claude Code Agent Skill

**Purpose:** Use Claude Code/Agent to intelligently assess environment and generate optimal `.env` configuration.

**Status:** Available as optional advanced workflow

---

## What This Does

Instead of manually copying .env.example and guessing configuration, you can have Claude Code:

1. ✅ Run `preflight-check.sh` (or preflight-check.bat) and capture output
2. ✅ Parse the environment assessment
3. ✅ Detect installed LLM backends (Ollama, LM Studio, vLLM, etc.)
4. ✅ Check Python keyring installation and configuration
5. ✅ Identify available secrets management (Keychain, Credential Manager, Secret Service)
6. ✅ Generate optimal `.env` file tailored to YOUR specific system
7. ✅ Provide recommendations for missing components
8. ✅ Test the configuration before setup

## What the Agent Will Do

### Step 1: Environment Assessment

The agent will:
- Detect your OS (macOS/Windows/Linux)
- Check for installed LLM backends
  - Ollama (primary)
  - LM Studio (alternative)
  - vLLM (Python-based)
  - llama.cpp
  - GPT4All
- Check Python environment
  - keyring (required)
  - keyrings.alt (Windows requirement)
- Verify secrets management setup
  - macOS: Keychain
  - Windows: Credential Manager (via keyrings.alt)
  - Linux: Secret Service (DBus)

### Step 2: Smart Configuration

Based on what's available, the agent will:

```
IF Ollama running
  → Use Ollama API endpoint
  → Detect models available
  → Suggest which model to use

IF LM Studio detected
  → Use LM Studio endpoint (port 1234)
  → Suggest configuration

IF vLLM installed
  → Use vLLM endpoint (port 8000)
  → Check for running instance

IF no LLM backend
  → Recommend Ollama
  → Provide installation link
  → Suggest fallback setup
```

### Step 3: Path Configuration

Based on OS:

```
macOS:
  THOTH_DATA_DIR=/Users/$(whoami)/thoth-data
  THOTH_WORKSPACE_DIR=/Users/$(whoami)/thoth-workspace

Windows (WSL2):
  THOTH_DATA_DIR=C:\Users\%USERNAME%\thoth-data
  THOTH_WORKSPACE_DIR=C:\Users\%USERNAME%\thoth-workspace

Linux:
  THOTH_DATA_DIR=/home/$(whoami)/thoth-data
  THOTH_WORKSPACE_DIR=/home/$(whoami)/thoth-workspace
```

### Step 4: Secrets Management

Optimal backend based on OS:

```
macOS:
  PYTHON_KEYRING_BACKEND=keyring.backends.macOS.Keyring

Windows:
  PYTHON_KEYRING_BACKEND=keyrings.alt.windows.CredentialVaultKeyring
  (requires: pip install keyrings.alt)

Linux:
  PYTHON_KEYRING_BACKEND=keyring.backends.secretservice.SecretServiceBackend
  (requires: apt install libsecret-1-dev)
```

## When to Use Each Approach

### Use `setup.sh` / `setup.bat`

**When:**
- You just want to get running quickly
- You don't need detailed recommendations
- You're comfortable with defaults

**Time:** ~1 minute

### Use `preflight-check.sh` / `preflight-check.bat`

**When:**
- You want to see what's available on your system
- You're troubleshooting setup issues
- You want to copy recommended config manually

**Time:** ~2 minutes

### Use Claude Code Agent (This)

**When:**
- You want intelligent, context-aware configuration
- You have a complex setup (multiple LLM backends)
- You want detailed recommendations
- You want everything automated and verified

**Time:** ~3-5 minutes (but more thorough)

## What Information Is Captured

The assessment collects:
- OS (macOS/Windows/Linux)
- Architecture (Intel/ARM/x86_64)
- Installed LLM backends
- Python version and packages
- Keyring configuration
- Secrets management availability
- Docker/docker-compose version
- Available utilities (curl, jq, git, nano)
- Environment variables

**Privacy note:** No personal data is captured. Only system configuration that you would normally see in terminal output.

## Troubleshooting

### "Ollama not detected but I have it installed"

```bash
# Check if Ollama is running
ps aux | grep ollama

# Check if it's in PATH
which ollama

# Add to PATH if needed
export PATH=$PATH:/path/to/ollama
```

### "keyring.alt not available on Windows"

```bash
# Install it
pip install keyrings.alt

# Verify
python -c "import keyrings.alt; print('OK')"
```

### "Secret Service not available on Linux"

```bash
# Install it
sudo apt install libsecret-1-dev

# Or for GNOME specifically
sudo apt install gnome-keyring
```
